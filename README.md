# <img src="https://upl.pp.ua/MiLauncher.svg" width="35" height="35" align="top"> MIUI Backup Layouts

Набор скриптов для сохранения и восстановления расположения иконок, папок, виджетов и других элементов рабочего стола MIUI (`com.miui.home`).

> 📁 **Экспорт:** Данные сохраняются в каталог рядом со скриптом по пути: `export/launcherXxX/`  
> 🗄️ **База данных:** Файл всегда один и находится по пути: `/data/user_de/0/com.miui.home/databases/launcherXxX.db`

---

## 📊 Структура базы данных

### Типы объектов (`itemType`)

| Значение | Описание |
| :---: | --- |
| **`0`** | Обычное приложение |
| **`1`** | Переключатели (Toggle shortcuts: Wi-Fi, Bluetooth и т.д.) |
| **`2`** | Папка |
| **`4`** | Виджет |
| **`14`** | Ярлыки активностей внутри приложения (Deep Shortcuts) |

### Расположение контейнеров (`container`)

| Значение | Описание |
| :---: | --- |
| **`-100`** | Объект лежит на рабочем столе (не в папке) |
| **`-101`** | Объект находится в док-баре (нижняя панель) |
| **`<ID папки>`** | Объект лежит внутри конкретной папки |

### Координаты и столы

* **`cellX` / `cellY`**: Координаты на сетке для ярлыков и папок. Нумерация начинается с **нуля**.  
    *Пример:* В зависимости от вашей сетки (например, 5x6), если вы хотите поместить объект в правый верхний угол, координаты будут `cellX=4`, `cellY=0`.
* **`screen=-1`**: Автоматическое смещение. Если объект сместится на другой стол и предыдущий останется пуст, система «минуснёт» его со сдвигом дальних. Если удалить ярлык внутри папки, остальные элементы также сместятся.

> **💡 На заметку (особенности ID):** > При удалении ярлыка или стола их ID в базе не уменьшается, так как используется `AUTOINCREMENT`. Например: у вас 4 стола с ID `1, 2, 3, 4`. Если удалить третий, в базе останутся столы с ID `1, 2, 4`.

## ⚙️ Особенности работы и восстановления

* **Мультисеточность:** Вы можете экспортировать бэкапы разных сеток (5x6, 8x10 и т.д.). После запуска восстановления скрипт предложит меню выбора конкретной сетки из сохраненных.
* **Адаптация:** Восстановление происходит в текущую активную сетку лаунчера, даже если экспортная база с ней не совпадает.
* **Частичное восстановление:** Реализована возможность точечного восстановления. При желании можно восстановить только папки или только виджеты.
